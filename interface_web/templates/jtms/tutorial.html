Introduction au JTMS

Bienvenue dans ce tutoriel interactif sur les Justification-based Truth Maintenance Systems !

Qu'est-ce qu'un JTMS ?

Un JTMS est un système qui maintient automatiquement la cohérence d'une base de connaissances en :

  • Gérant les croyances : ce que le système "croit" être vrai
  • Traçant les justifications : pourquoi il croit que c'est vrai
  • Propageant les changements : mise à jour automatique
  • Détectant les contradictions : résolution des conflits
Exemple Concret

Imaginez un système qui doit déterminer si "Tweety peut voler" :

Croyances et Justifications

Les Croyances (Beliefs)

Une croyance représente une proposition que le système considère comme vraie, fausse, ou inconnue.

VRAIE

Le système croit que c'est vrai

FAUSSE

Le système croit que c'est faux

INCONNUE

Le système ne sait pas

Les Justifications

Une justification explique pourquoi une croyance est vraie ou fausse.

Construction d'une Justification
REGLE: Si A et B alors C
A est vrai + B est vrai → C est vrai

Types de Justifications

Classification des Justifications

1. Justifications de Support (SL)

Listent les croyances qui supportent la conclusion.

SL("Tweety vole") = ["Tweety oiseau", "Oiseaux volent"]
2. Justifications de Négation (NSL)

Listent les croyances qui contredisent la conclusion.

NSL("Tweety vole") = ["Tweety pingouin"]

Propagation des Croyances

Mécanisme de Propagation

Quand une croyance change, le JTMS met automatiquement à jour toutes les croyances dépendantes.

  1. Détection du changement : Une croyance passe de vraie à fausse
  2. Identification des dépendances : Quelles autres croyances en dépendent ?
  3. Réévaluation : Recalcul des statuts des croyances dépendantes
  4. Propagation récursive : Le processus continue en cascade
